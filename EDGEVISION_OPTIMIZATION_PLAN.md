## 二、核心策略：重新定位

### ❌ 别再用这些说法
- ~~"可配置化 Pipeline，零代码搭建检测流程"~~（你没做到，评审一眼看穿）
- ~~"多模态算法融合"~~（三个独立功能拼一起不叫融合）

### ✅ 改用这些表述

| 原说法 | 改为 |
|--------|------|
| 可配置化 Pipeline | **参数化视觉检测流水线** — 9 步处理链路，每步参数实时可调，所见即所得 |
| 多模态融合 | **同平台覆盖三种检测能力** — 传统图像处理 / 深度学习 / 条码识别一站式完成 |
| 零代码搭建 | **免编程参数化配置** — 滑块+下拉+勾选即可完成检测方案设定 |
| 边云协同 | **MQTT 5.0 边云协同** — 边缘端实时推理 + 云端可视化管控闭环 |

**一句话定位（建议参赛用）：**
> EdgeVision：面向中小型制造企业的**参数化边云协同视觉检测平台**，覆盖传统图像处理、深度学习推理与条码识别三种检测能力，通过 MQTT 5.0 实现边缘端实时检测与云端远程管控的完整闭环。

## 三、实施路线图（分 5 个 Sprint）

### Sprint 1：补短板（第 1-2 周）

#### 1.1 真正实现 Pipeline 步骤级可配置（最重要）

当前问题：`initPipeline()` 硬编码 9 步，无法按需启用/禁用/排序。

需要改动：

```
pipeline_manager.cpp:174-187  ← 核心改这里
```

**实现方式（单人 2 周内可完成）：**

```
1. 在 PipelineConfig 中增加 stepEnabled flags
   struct PipelineConfig {
       bool stepEnabled[9] = {true,...};  // 每步是否启用
       int stepOrder[9] = {0,1,2,...};    // 步骤执行顺序
       ...
   };

2. initPipeline() 按 stepOrder 和 stepEnabled 动态构建
   void PipelineManager::initPipeline() {
       m_pipeline = Pipeline();
       // 注册所有步骤到 map
       std::map<int, std::unique_ptr<IPipelineStep>> allSteps;
       allSteps[0] = std::make_unique<StepColorChannel>();
       allSteps[1] = std::make_unique<StepEnhance>(m_processor.get());
       ...
       // 按配置顺序添加
       for (int idx : m_config.stepOrder) {
           if (m_config.stepEnabled[idx] && allSteps.contains(idx)) {
               m_pipeline.add(std::move(allSteps[idx]));
           }
       }
   }

3. 在 UI 上添加步骤勾选框（简单，各 Tab 加个 QCheckBox）
```

**预计工作量：** 约 200 行改动，3-4 天单人可完成。

**如果不能做拖拽排序 UI：**
- 最低方案：加 9 个 `QCheckBox` 让用户勾选启用/禁用步骤
- 这已经能支撑"步骤可配置"的说法
- 排序用固定顺序即可，不必做拖拽

#### 1.2 打通 YOLO → Pipeline 融合（关键亮点）

当前问题：YOLO 在独立 Tab 跑，Pipeline 在另一个 Tab 跑，互不关联。

**实现方式：**

```
1. 在 PipelineConfig 中增加：
   struct PipelineConfig {
       bool enableObjectDetection = false;  // 在 Pipeline 中启用 YOLO
       QString onnxModelPath;               // ONNX 模型路径
       float confidenceThreshold = 0.5;
       float nmsThreshold = 0.45;
       ...
   };

2. 新增 StepObjectDetection : IPipelineStep
   - 在 Pipeline 中插入，可选位置（如步骤 6 之后）
   - 将 YOLO 检测结果叠加到 ctx.visualBase
   - 检测结果写入 ctx 供后续步骤使用

3. 复用现有的 OrtInference（已实现 501 行）
```

**预计工作量：** 约 150 行，2-3 天。

**效果：** 可以展示"传统处理+深度学习在一条 Pipeline 中协同工作"，这是真正的多模态融合。
